# Telegram Finance Tracker Bot

A multi-user Telegram bot that logs casual expense messages, stores them in Supabase, answers spending summary questions, supports `undo`, and sends an end-of-month report automatically.

## Project Structure

```text
telegram-finance-tracker/
├── app/
│   ├── __init__.py
│   ├── claude_handler.py
│   ├── config.py
│   ├── database.py
│   ├── scheduler.py
│   └── telegram_handler.py
├── .env.example
├── main.py
├── README.md
└── requirements.txt
```

## Features

- Webhook-based Telegram bot for Railway deployment
- Natural-language expense logging through Claude
- Multi-user storage keyed by Telegram user ID
- Summary requests like `how much did I spend this week?`
- `undo` support to delete the latest expense
- Automatic monthly summaries on the last day of the month
- Default currency set to Philippine Peso (`PHP` / `₱`)

## 1. Create the Telegram Bot

1. Open Telegram and search for [@BotFather](https://t.me/BotFather).
2. Send `/newbot`.
3. Choose a display name for the bot.
4. Choose a unique bot username ending in `bot`.
5. Copy the bot token that BotFather gives you.
6. Optional but helpful: use `/setdescription` and `/setuserpic` to make the bot clearer for users.

Save that token for `TELEGRAM_BOT_TOKEN`.

## 2. Set Up Supabase

Create a new Supabase project, then open the SQL editor and run this:

```sql
create table if not exists public.expenses (
  id bigint generated by default as identity primary key,
  telegram_user_id bigint not null,
  telegram_username text,
  item text not null,
  amount numeric(12, 2) not null check (amount > 0),
  category text not null check (
    category in (
      'food',
      'transport',
      'groceries',
      'bills',
      'shopping',
      'health',
      'entertainment',
      'other'
    )
  ),
  currency text not null default 'PHP',
  created_at timestamptz not null default timezone('utc', now())
);

create index if not exists expenses_telegram_user_id_idx
  on public.expenses (telegram_user_id);

create index if not exists expenses_created_at_idx
  on public.expenses (created_at desc);
```

Then copy:

- `SUPABASE_URL` from Project Settings
- `SUPABASE_KEY` using the service role key so the bot can write and delete rows safely on the server side

## 3. Local Setup

1. Create a folder for the project and move into it:

   ```bash
   cd telegram-finance-tracker
   ```

2. Create and activate a virtual environment:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. Install dependencies:

   ```bash
   pip install -r requirements.txt
   ```

4. Copy `.env.example` to `.env` and fill in your real values.

5. Run the app locally:

   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

## 4. Environment Variables

Add these in Railway:

- `TELEGRAM_BOT_TOKEN`
- `ANTHROPIC_API_KEY`
- `ANTHROPIC_MODEL`
- `SUPABASE_URL`
- `SUPABASE_KEY`
- `APP_BASE_URL`
- `TELEGRAM_WEBHOOK_SECRET`
- `TIMEZONE`
- `MONTHLY_SUMMARY_HOUR`
- `MONTHLY_SUMMARY_MINUTE`

Recommended values:

- `ANTHROPIC_MODEL=claude-haiku-4-5`
- `TIMEZONE=Asia/Manila`
- `MONTHLY_SUMMARY_HOUR=21`
- `MONTHLY_SUMMARY_MINUTE=0`

## 5. How Claude Parsing Works

Every non-command text message is sent to Claude, which returns JSON only. The bot expects these intents:

- `expense`
- `summary`
- `undo`
- `unknown`

For expense messages, Claude extracts:

- `item`
- `amount`
- `category`
- `currency`

If a message is unclear or missing details, Claude marks it for clarification and the bot asks the user to retry instead of storing bad data.

## 6. Deploy to Railway

1. Push this project to a GitHub repository.
2. In Railway, create a new project.
3. Choose `Deploy from GitHub repo`.
4. Select the repository containing this bot.
5. Add all environment variables from the previous section.
6. Railway should detect Python automatically.
7. Set the start command to:

   ```bash
   uvicorn main:app --host 0.0.0.0 --port $PORT
   ```

   Or let Railway use the included `Procfile`.

8. After the deployment finishes, copy the public Railway URL.
9. Set `APP_BASE_URL` to that exact public URL and redeploy if needed.

## 7. Set the Telegram Webhook

If `APP_BASE_URL` is set, the app will try to register the webhook automatically at startup.

The webhook URL format is:

```text
https://your-app.up.railway.app/webhook/your-secret
```

If you want to set it manually, open this URL in your browser after replacing the placeholders:

```text
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://your-app.up.railway.app/webhook/<TELEGRAM_WEBHOOK_SECRET>
```

To verify it:

```text
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getWebhookInfo
```

## 9. Notes

- The bot is built for server hosting and uses webhook mode, not polling.
- Monthly reports are sent to users who already have at least one saved expense.
- The scheduler runs inside the Railway app process, so the service should stay deployed continuously.
- For production use, keep your Supabase service role key private and only on the server.
